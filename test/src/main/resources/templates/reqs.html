Requisitos Funcionales:

  • El sistema debe permitir a los usuarios buscar propiedades para poder solicitar el arriendo en fechas específicas.
  • El sistema debe permitir a los usuarios activar sus cuentas para que puedan acceder a las herramientas y recursos necesarios para realizar las actividades del sistema.
  • El sistema debe permitir a los usuarios crear una cuenta en el sistema para manejar propiedades, de manera que puedan acceder a las herramientas y recursos necesarios para realizar las actividades del sistema.
  • El sistema debe permitir a los usuarios cancelar la solicitud de arriendo.
  • El sistema debe permitir a los usuarios aceptar la solicitud de arriendo.
  • El sistema debe permitir a los usuarios ver las solicitudes de arriendo para aceptarlas o rechazarlas.
  • El sistema debe permitir a los usuarios desactivar propiedades para que no les arrienden propiedades que ya no tienen en su disposición.
  • El sistema debe permitir a los usuarios autenticarse en el sistema para realizar el paso previo para pagar el posible arriendo.
  • El sistema debe permitir a los usuarios autenticarse en el sistema para verificar sus solicitudes de arriendo.
  • El sistema debe permitir a los usuarios editar sus propiedades para poder arrendarlas y actualizarlas.
  • El sistema debe permitir a los usuarios autenticarse en el sistema para pagar los arriendos.
  • El sistema debe permitir a los usuarios calificar a los arrendatarios para poder verificar y mostrar qué tan buenos son los arrendatarios.
  • El sistema debe permitir a los usuarios crear propiedades para poder arrendarlas.
  • El sistema debe permitir a los usuarios ver las propiedades que tienen a su cargo para poder realizar las acciones que el sistema permite.

Requisitos No Funcionales:

  • El sistema debe permitir a los usuarios autenticarse en el sistema para poder acceder de forma segura a sus cuentas.
  • El sistema debe permitir a los usuarios subir fotos de las propiedades para poder mostrar las propiedades de mejor forma.
